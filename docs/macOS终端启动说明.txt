TRPG模组笔记 macOS 终端启动说明

如果双击 .app 后没有成功启动，可以按下面步骤在终端中手动启动。
这是一份给完全不懂命令行的用户准备的教程，请尽量照着做，不要自己改字。

一、先确认下载了正确版本

1. Apple Silicon 芯片（M1 / M2 / M3 / M4 等）请下载 darwin arm64 版本
2. Intel 芯片 Mac 请不要使用当前这个包
3. 若启动时出现 Bad CPU type in executable，通常就是下载了错误架构的版本

二、系统版本要求

1. 建议使用较新的稳定版 macOS
2. 当前打包配置的最低系统版本为 macOS 11.0

三、先把文件放到一个你找得到的地方

请先做这几件事：

1. 打开“访达”
2. 打开“下载”文件夹
3. 找到你下载的压缩包并解压
4. 解压后，你应该能看到一个文件夹，名字通常是：

TRPG模组笔记-macOS

5. 打开这个文件夹后，里面应该能看到：

TRPG模组笔记.app
macOS终端启动说明.txt

如果你现在看不到这两个东西，请先不要继续，说明你打开的地方不对。

四、打开终端

1. 按键盘上的 Command + 空格
2. 输入“终端”
3. 打开“终端”应用

打开后会看到一个黑色或白色窗口，这就是终端。

五、不要自己手打命令，直接复制下面这一整段

特别注意：

1. 不要把命令拆开输入
2. 不要自己删空格
3. 不要自己改大小写
4. 特别注意是 MacOS，不是 MacOs
5. 特别注意是 chmod +x，不是 chmod+x
6. 特别注意是 xattr -dr，不是 xattr-dr

请复制下面整段命令，一次性粘贴到终端里，然后按回车：

cd ~/Downloads
ls -lah
cd ~/Downloads/"TRPG模组笔记-macOS"
ls -lah
ls -lah "./TRPG模组笔记.app/Contents/MacOS"
chmod +x "./TRPG模组笔记.app/Contents/MacOS/trpg-note"
chmod +x "./TRPG模组笔记.app/Contents/MacOS/trpg-note-launcher"
xattr -dr com.apple.quarantine "./TRPG模组笔记.app"
"./TRPG模组笔记.app/Contents/MacOS/trpg-note-launcher"

六、你应该看到什么

正常情况下，前面几行会显示很多文件名，其中应该能看到：

TRPG模组笔记-macOS
TRPG模组笔记.app

然后程序会尝试启动。

七、启动成功后做什么

打开浏览器，访问：

http://localhost:8080/web

八、如果终端提示没有这个文件

如果你看到下面这种提示：

No such file or directory

这通常不是程序坏了，而是下面几种情况之一：

1. 你没有先把压缩包解压
2. 你打开的不是“下载”文件夹
3. 解压出来的文件夹不叫 `TRPG模组笔记-macOS`
4. 你把 `.app` 单独拖走了，和说明文件不在一起
5. 你复制命令时漏了一部分

这时候请回到第三步，重新确认：

1. 你打开的是不是“下载”文件夹
2. 你能不能看到 `TRPG模组笔记-macOS`
3. 进入那个文件夹后，能不能看到 `TRPG模组笔记.app`

九、如果还是不行，请把这些内容发给开发者

请把终端里看到的全部内容截图，或者复制给开发者。

如果你会复制命令，也请再执行下面这几行，并把输出一起发来：

pwd
ls -lah
cd ~/Downloads
ls -lah
cd ~/Downloads/"TRPG模组笔记-macOS"
ls -lah
ls -lah "./TRPG模组笔记.app/Contents/MacOS"

如果程序已经启动了，但网页还是打不开，也请把终端里最后几行输出一起发给开发者。
