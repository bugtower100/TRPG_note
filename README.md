# TRPG_note
如果这个项目对你有帮助，欢迎点个 Star⭐ 支持一下！

为方便各位跑团 GM 备团所作的本地笔记软件，目标是让信息整理更快、检索更准、跑团时更稳。

## 项目简介

TRPG_note 是一款面向主持人（GM）的模组管理工具。  
你可以用它集中管理角色、怪物、地点、组织、事件、线索和时间线，并通过富文本联动快速跳转到相关资料。

适用场景：
- 长模组（多势力、多线索）
- 即兴扩展（随时新增自定义设定）
- 复盘整理（按时间线回看事件）

## 功能总览

### 1) 多模组管理
- 创建、进入、切换、删除多个模组
- 每个模组互相独立，避免数据混杂
- 支持导出单模组 JSON，便于分享与备份

### 2) 实体管理（核心资料库）
- 支持实体类型：角色 / 怪物 / 地点 / 组织 / 事件 / 线索 / 时间线
- 每个实体都支持基础信息与详细描述
- 列表页支持快速筛选与检索

### 3) 详情页区块编辑（重点）
- 内置区块：按常见备团结构预置
- 自定义区块：可按你的风格补充设定
- 支持持续迭代，不会被固定模板限制

### 4) 富文本联动
- 在文本中识别关键实体
- 点击后可直接打开关联内容
- 支持并行标签查看，减少来回切页

### 5) 时间线管理
- 事件节点按时间组织
- 便于把“线索出现顺序、NPC行动、世界变化”串起来
- 适合控制节奏与伏笔回收

### 6) 主题与体验
- 支持多主题切换（默认、羊皮纸、未来、自然等）
- 适配长时间备团阅读

### 7) 数据能力
- JSON 导入 / 导出
- 本地文件加载与保存
- 支持坏数据清理与结构修复（维护命令）

## 下载与安装

### 普通用户（推荐）
1. 打开项目的 GitHub Releases 页面
2. 下载最新版本 `TRPG模组笔记.exe`
3. 将 EXE 放到你常用目录（如 `D:\TRPG`）
4. 双击启动即可使用

### 便携建议
- 建议把 EXE 与备份文件放在同一个专用目录
- 不建议放在系统保护目录（如 `C:\Program Files`）

## 使用说明（快速上手）

### 第一步：创建模组
进入首页后先创建一个模组，填写名称与简介。

### 第二步：录入核心实体
先录入 3 类最关键信息：
- 重要 NPC（角色）
- 关键地点
- 核心事件

### 第三步：补充线索与时间线
- 给每个事件补上线索来源
- 把关键节点放入时间线

### 第四步：跑团前检查
- 导出一份 JSON 备份
- 检查角色与组织关系是否完整
- 检查时间线是否有冲突

## 开发与维护

### 环境要求
- Node.js 18+、npm 9+（前端构建）
- Go 1.21+（后端服务）
- Windows（当前打包目标）

### 常用命令

```bash
npm install
npm run dev
npm run build
npm run lint
npm run check
go run ./server
```

### 构建与发布

```bash
# 1) 构建前端并同步到 Go 内置静态目录
npm run build

# 2) 构建后端可执行文件
cd server && go build -o release/TRPG模组笔记.exe
```

运行方式：
- 双击 `release/TRPG模组笔记.exe` 后，访问托盘弹出的地址或浏览器自动打开的页面
- 若需调试 API，仅运行后端：`go run ./server`，前端开发服：`npm run dev`（已配置 `/api` 代理至 `:8080`）

## 常见问题（FAQ）

### Q1：双击 EXE 后浏览器没打开怎么办？
- 先确认安全软件没有拦截本地启动
- 再次启动 EXE，等待几秒
- 如仍失败，尝试管理员身份运行一次

### Q2：如何退出程序？
- 通过系统托盘图标退出，或直接关闭后台进程

### Q3：数据存在哪里？
- 默认保存在 `server/data/storage.db`（SQLite）
- 建议定期导出 JSON 做额外备份

### Q4：如何迁移到新电脑？
0. 直接把data目录和server.exe拷贝到新电脑
1. 在旧电脑导出 JSON
2. 新电脑安装并启动应用
3. 使用导入功能恢复数据
4. 未来会增加基于云同步的模式！

### Q5：出现异常或数据损坏怎么办？
- 先备份当前数据文件
- 使用项目维护命令进行清理修复
